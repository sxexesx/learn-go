# Введение. Основные понятия.

## Почему появился GO

1. Cложность существующих языков с чрезмерно синтаксической строгостью и сложностью
2. Медленная компиляция: крупные проекты на существующих языках требовали долгого времени сборки
3. Проблемы с многозадачностью: ресурсоемкие решения для потоков снижали эффективность распределенных систем
4. Изменился вектор направленности взаимодействия приложений в сторону межсетевого
5. Поддержка современных архитектур - микросервисный подход


## Основные цели:

1. Эффективен как статически-типизируемый ЯП и легок как динамический ЯП
2. Type-Safe и Memory-Safe
3. Поддержка конкурентности
4. Сборка мусора без задержек
5. Высокая скорость компиляции


## Основные недостатки языка GO

1. Отсутствие перегрузки функций (одинаковое название, но разное количество аргументов)
2. Отсутствие параметров по умолчанию
3. Обработка ошибок - постоянно встречается одна и та же конструкция обработки
4. Многословность кода
5. Нет библиотеки для GUI
6. Ограниченная область применения: не подходят для ML, научных вычислений, игр, скриптов.
7. "Магический" сборщик мусора


## Основные утилиты пакета go

`go build` - компилирует пакеты и их зависимости в исполняемый файл.    

`go run` - компилирует и сразу же запускает программу в одном шаге. Создает временный исполняемый файл, который удаляется сразу после завершения  

`go install` - компилирует и устанавливает пакеты в директори, указанную в переменной окружения GOBIN

`go test` - тестирование пакетов. Ищет файлы с суффиксом `_test.go` и запускает функции вида: TestXXX().  
    Флаги:   
    `-v` - подробный вывод,  `-cover` - анализ покрытия, `-bench` - бенчмарки, `-timeout 0` - без таймаута

`go clean` - удаляет объектные файлы и кэшированные файлы, оставленные другими инструментами.
    Флаги:
    `-cache`, `testcache`, `-modcache` - соответствующие кэши

`go get` - добавляет зависимости в текущий модуль, устанавливает их и обновляет go.mod

`go mod` - управление модулем через подкоманды:
    - `init` - инициализация нового модуля  
    - `tidy` - удаляет неиспользуемые зависимости и добавляет недостающие  
    - `download` - загружает модули в локальный кэш  
    - `vendor` - создает клопию зависимостей в директории `vendor`  
    - `graph` - выводит граф зависимостей модуля  
    - `verify` - проверяет, что загруженные пакеты не были изменены  
    - `why` - объясняет почему конкретный пакет нужен проекту  
    - `edit` - редактирует go.mod программно

`go list` - выводит информацию о пакетах или модулях, полезно для получения структурированной информации  

## Форматирование и анализ

`go fmt` - форматирует код согласно стандартам Go

`go vet` - анализирует код в поиске потенциальных ошибок и подазрительных паттернов

`go fix` - автоматически обновляет пакеты для использования новых API. Применяется при обновлении Go версии для миграции кода

## Документация и информация

`go doc ` - выводит документацию для пакета или функции. Флаг `-http :6060` запускает HTTP сервер на 6060 порту

`go env` - печатает информацию об окружении Go

## Генерация и запуск

`go generate` - запускает команды генерации в специальных комментариях с директивами //go:generate

`go tool` - запускает указанный Go инструмент. Без аргументов выводит список доступных инструментов.

`go version` - покажет текущую версию Go или на какой версии был собран бинарник

`go bug` - открывает браузер для создания отчета об ошибке с автоматически собранной информацией о системе

## Управление рабочей областью  
`go work` - управление Go workspaces через подкоманды:
    - `init` - инициализирует файл `go.work`  
    - `use` - добавляет модули в рабочую область  
    - `edit`- редактирует `go.work` программно  
    - `sync` - синхронизирует список сборки с модулями  
    - `vendor` - создает копию зависимостей  

## Телемитрия (1.23+)

`go telemetry` - управляет данными телеметрии, которые Go может собрать для улучшения. Позволяет включать отключать сбор:
    - `-v` - подробный вывод  
    - `-x` - выводит выполняемые команды  
    - `-n` - выводит команды без выполнения  
    - `-race` - включает детектор гонок данных  
    - `-cover` - анализ покрытия кода  
    - `-tags` - условная компиляции  
    - `ldflags` - флаги для компановщика  

## Основные команды языка go

```
go mod init <package_name> - инициализация модулей в проекте
go mod tidy - подтянуть зависимости в проекте, которых нет


go run <path_to_main.go> - компиляция и запуск

go get github.com/.... - используется для получения, добавления или обновления зависимостей в проекте (файл go.mod)
go install - установка в проект бинарных файлов в проект. Не изменяет go.mod

go fmt path/to/your/package - форматирование кода

go test  - запуск тестов без таймаута
```
